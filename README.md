# Exercise 2.15 — CsiNet Generalization on Multiple COST 2100 Channel Datasets

本專案對應 *Wireless Communications and Machine Learning* Exercise 2.15。目標是測試原本訓練好的 CsiNet 在多組不同 COST 2100 channel datasets 上的 CSI reconstruction NMSE，並進一步使用混合資料集重新訓練 CsiNet，觀察模型在不同通道分佈下的泛化能力。

本資料夾維持原始 Exercise 2.15 的檔案配置，只將原始 `.txt` 程式檔改為可直接執行的 `.py` 檔：

```text
CS-CsiNet_onlytest.py
CS-CsiNet_train.py
CsiNet_onlytest.py
CsiNet_train.py
README.md
```

其中本題主要修改 `CsiNet_onlytest.py` 與 `CsiNet_train.py`。`CS-CsiNet_train.py` 與 `CS-CsiNet_onlytest.py` 保留作為原始 repository 的完整檔案。

---

## 1. Assignment Description

Exercise 2.15 asks the following:

1. Use the COST 2100 channel model to generate more than five different channel datasets, for example by changing user distributions.
2. Evaluate the CSI reconstruction NMSE of a trained CsiNet model on each dataset.
3. Mix the above channel datasets and use them to train CsiNet, then compare the reconstruction performance with the results in step 2.

The main purpose is to evaluate whether CsiNet trained on one channel distribution can generalize to other channel distributions, and whether mixed-distribution training improves robustness.

---

## 2. Modified Files

### `CsiNet_onlytest.py`

This script corresponds to part (b). It loads a trained CsiNet model and evaluates it on multiple COST 2100 datasets.

Added functions:

- Evaluate more than five test datasets in one run.
- Compute NMSE for each dataset.
- Compute correlation coefficient if the corresponding frequency-domain `HF_all` file is available.
- Save all evaluation results as a CSV file.

Output example:

```text
result/ex215_eval_pretrained_CsiNet_indoor_dim512.csv
```

### `CsiNet_train.py`

This script corresponds to part (c). It mixes multiple COST 2100 datasets and trains CsiNet using the combined training set.

Added functions:

- Load multiple training and validation datasets.
- Concatenate them into one mixed training dataset.
- Train CsiNet using the mixed dataset.
- Evaluate the mixed-trained model on every individual test dataset.
- Save the mixed-training evaluation result.
- If the baseline CSV from part (b) exists, automatically generate a comparison CSV.

Output examples:

```text
result/ex215_eval_mixed_CsiNet_indoor_dim512_MM_DD.csv
result/ex215_compare_CsiNet_indoor_dim512_MM_DD.csv
```

## 3. Dataset Setting

The default experiment uses six COST 2100 datasets with different user distributions:

```text
user_center
user_edge
user_uniform
user_left_cluster
user_right_cluster
user_ring
```

Each dataset should be generated by COST 2100 and saved in MATLAB `.mat` format.

For indoor datasets, the expected file naming rule is:

```text
data/DATA_Htrainin_<dataset_name>.mat
data/DATA_Hvalin_<dataset_name>.mat
data/DATA_Htestin_<dataset_name>.mat
data/DATA_HtestFin_<dataset_name>_all.mat
```

For example:

```text
data/DATA_Htrainin_user_center.mat
data/DATA_Hvalin_user_center.mat
data/DATA_Htestin_user_center.mat
data/DATA_HtestFin_user_center_all.mat
```

Each time-domain CSI file should contain:

```text
HT
```

The optional frequency-domain evaluation file should contain:

```text
HF_all
```

## 5. How to Run

### Step 1: Prepare datasets

Generate more than five COST 2100 channel datasets by changing the user distribution or other channel parameters, and place the generated `.mat` files in the `data/` folder.

### Step 2: Prepare pretrained CsiNet model

Place the pretrained CsiNet model files in `saved_model/`:

```text
saved_model/model_CsiNet_indoor_dim512.json
saved_model/model_CsiNet_indoor_dim512.h5
```

The `envir` and `encoded_dim` settings in the script must match the pretrained model.

### Step 3: Evaluate pretrained CsiNet on multiple datasets

```bash
python CsiNet_onlytest.py
```

This produces the part (b) baseline result:

```text
result/ex215_eval_pretrained_CsiNet_indoor_dim512.csv
```

### Step 4: Train CsiNet using mixed datasets

```bash
python CsiNet_train.py
```

This trains CsiNet using all selected datasets and evaluates the trained model on each individual test set.

---

## 6. Evaluation Metric

The main metric is NMSE in dB:

```text
NMSE(dB) = 10 log10( E[|H - H_hat|^2] / E[|H|^2] )
```

A smaller NMSE is better. In dB scale, a more negative value means better CSI reconstruction quality.

---

## 7. Discussion

If CsiNet is trained only on one channel distribution, its reconstruction performance may degrade when tested on other channel distributions. This indicates that the model learns distribution-specific channel characteristics.

After mixing several COST 2100 datasets for training, the model usually observes more diverse channel samples. Therefore, its reconstruction performance on unseen or shifted channel distributions can become more stable. This suggests that dataset diversity, domain randomization, and mixed-scenario training are useful methods to improve the generalization ability of CSI feedback networks in practical wireless systems.
